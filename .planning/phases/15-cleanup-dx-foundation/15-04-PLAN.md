**Step 1: Add validateCredentials function to src/index.ts**
Add this function BEFORE the `start()` function (around line 450):

```typescript
/**
 * Validate required environment variables before server startup.
 * Exits with a clear, actionable error if credentials are missing.
 */
function validateCredentials(): void {
  const email = process.env.ZLIBRARY_EMAIL;
  const password = process.env.ZLIBRARY_PASSWORD;

  if (!email || !password) {
    const missing: string[] = [];
    if (!email) missing.push('ZLIBRARY_EMAIL');
    if (!password) missing.push('ZLIBRARY_PASSWORD');

    console.error(
      `\nError: Missing required environment variable(s): ${missing.join(', ')}\n\n` +
        `Z-Library credentials are required for the MCP server to function.\n\n` +
        `To fix this, set the following in your MCP client configuration:\n` +
        `  "env": {\n` +
        `    "ZLIBRARY_EMAIL": "your-email@example.com",\n` +
        `    "ZLIBRARY_PASSWORD": "your-password"\n` +
        `  }\n\n` +
        `For Claude Desktop, add this to your claude_desktop_config.json.\n` +
        `For other MCP clients, consult your client's documentation.\n` +
        `See README.md for detailed setup instructions.\n`,
    );
    process.exit(1);
  }
}
```

**Step 2: Call validateCredentials() at the start of the start() function**
Inside the `start()` function, add `validateCredentials()` as the FIRST line inside the try block (before the logs directory creation), but ONLY when not in testing mode:

```typescript
async function start(opts: StartOptions = {}): Promise<...> {
  try {
    // Validate credentials before anything else (skip in test mode)
    if (!opts.testing) {

    }

    // Ensure the logs directory exists
```

The `opts.testing` guard is critical — tests mock the Python bridge and don't need real credentials. Without this guard, every Jest test would fail.

Notes:

- `coverageReporters`: `text` for terminal output, `lcov` for CI integration (Phase 17), `json-summary` for programmatic access
- `collectCoverageFrom`: targets compiled output in dist/ (Jest runs JS, not TS)
- Thresholds from research: 76.86% stmts, 71.06% branches, 61.25% functions, 79.16% lines → minus 5% each

**Step 4: Configure pytest coverage in pytest.ini**
IMPORTANT: The project has a `pytest.ini` file at the root. Pytest's config priority is `pytest.ini > pyproject.toml`, so `[tool.pytest.ini_options]` in `pyproject.toml` is NOT consulted when `pytest.ini` exists. All pytest runtime config must go in `pytest.ini`.

Update the `addopts` line in `pytest.ini` to include coverage flags:

Add `--cov=lib --cov-report=term-missing --cov-fail-under=XX` to the existing addopts (where XX is the measured baseline minus 5, rounded down to nearest integer).

For example, if pytest reports 85% coverage, set `--cov-fail-under=80`.

Preserve existing addopts flags (`--strict-markers`, `--benchmark-disable` if present). The final line in `pytest.ini` should look like:
```

```

Note: `uv add --dev pytest-cov` correctly adds the library to `pyproject.toml` dev-dependencies. Only the pytest runtime config (addopts) goes in `pytest.ini`.
