**Step 1: Install devDependencies**

npm install --save-dev eslint @eslint/js typescript-eslint eslint-config-prettier prettier

**Step 2: Create eslint.config.js**
Create `eslint.config.js` in project root (flat config format, `.js` extension because `"type": "module"`):

```javascript
import eslint from '@eslint/js';
import tseslint from 'typescript-eslint';
import eslintConfigPrettier from 'eslint-config-prettier/flat';

export default tseslint.config(
  eslint.configs.recommended,
  tseslint.configs.recommended,
  eslintConfigPrettier,
  {
    rules: {
      '@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }],
      '@typescript-eslint/consistent-type-imports': 'error',
      '@typescript-eslint/no-explicit-any': 'off',
    },
  },
  {
    ignores: ['dist/', 'node_modules/', 'coverage/', '__tests__/', '__mocks__/', 'scripts/', 'lib/', 'zlibrary/'],
  },
);
```

Notes on rules per user decision:
- `@typescript-eslint/recommended` as base (LOCKED)
- `no-unused-vars` with `argsIgnorePattern: '^_'` — allows underscore-prefixed unused args (common pattern in this codebase for callback params like `_args`)
- `consistent-type-imports` — enforces `import type` syntax (LOCKED per DX-01)
- `no-explicit-any: off` — the codebase uses `any` extensively in the Python bridge layer and MCP tool handlers; enforcing this would require hundreds of changes out of scope

**Step 5: Update lint-staged in package.json**
Change the lint-staged configuration from:
```json
"lint-staged": {
  "src/**/*.{ts,js}": "bash -c 'tsc --noEmit'",

  ]
}
```

To:
```json

  "src/**/*.ts": [

  "*.py": [
    "uv run ruff check --fix",
    "uv run ruff format"
  ]
}
```

Note: Changed glob from `src/**/*.{ts,js}` to `src/**/*.ts` since .js files in src/ are now gitignored (Plan 02). Order: ESLint first (logic), Prettier second (formatting), tsc last (type-check final output). Per user decision, ordering is at Claude's discretion.
