<task type="auto">
  <name>Task 1: Configure npm files whitelist and verify tarball</name>
  <files>package.json, .npmignore</files>
  <action>
Add `files` field to package.json with the following whitelist (per research findings -- these are the runtime-required files totaling ~2.7MB):

```json
"files": [
  "dist/",
  "lib/",
  "zlibrary/",
  "pyproject.toml",
  "uv.lock",
  "setup-uv.sh"
]
```

Place the `files` field after the `license` field (before `dependencies`) for conventional ordering.

npm always includes package.json, README.md, CHANGELOG.md, and LICENSE regardless of the `files` field.

After adding the field, verify with `npm pack --dry-run` that:
1. Total file count is approximately 15-25 files (not 985)
2. Total size is under 5MB (target ~2.7MB uncompressed, ~1.5MB compressed)
3. The output includes: dist/index.js, dist/lib/*.js, lib/python_bridge.py, lib/rag_processing.py, zlibrary/ contents, pyproject.toml, uv.lock, setup-uv.sh
4. The output does NOT include: __tests__/, test_files/, .planning/, claudedocs/, .claude/, scripts/, src/, docs/, coverage/, __pycache__/

Also verify that the existing `.npmignore` already has `__pycache__/` (per research, it does) so that __pycache__ dirs within `lib/` are excluded even though `lib/` is whitelisted. If `.npmignore` does not have `__pycache__/`, add it.

If `.dockerignore` does not exist or is incomplete, create/update it to exclude test files, docs, .planning/, claudedocs/, .claude/, coverage/, src/, __tests__/, test_files/, scripts/, node_modules/, .venv/, .git/ to keep Docker build context small.
  </action>
