Fix the test at line 194 in `__tests__/zlibrary-api.test.js`.

The test "should throw error if Python script returns non-JSON string" expects a specific error message format that changed in Node 22.

**Change line 194 from:**
```javascript
.toThrow(/Failed to parse initial JSON output from Python script: Unexpected token T in JSON at position 0/);
```

**To:**
```javascript
.toThrow(/Failed to parse initial JSON output from Python script: Unexpected token/);
```

This matches on the stable prefix of the error message, which is the same across all Node versions. The varying suffix (old: `"T in JSON at position 0"`, new: `"'T', \"This is not JSON\" is not valid JSON"`) is not load-bearing for the assertion.
