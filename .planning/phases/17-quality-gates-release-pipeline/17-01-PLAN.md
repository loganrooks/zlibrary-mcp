**1. Composite action at `.github/actions/setup/action.yml`:**

Extract the repeated setup steps (checkout, setup-node, setup-uv, npm ci, uv sync, optional build) into a reusable composite action. This deduplicates the 6+ identical setup blocks across CI jobs.

Structure:
```yaml
name: 'Project Setup'
description: 'Checkout, install Node.js 22, UV, and all dependencies'
inputs:
  node-version:
    description: 'Node.js version'
    default: '22'
  build:
    description: 'Run npm build after install'
    default: 'true'
runs:
  using: 'composite'
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: ${{ inputs.node-version }}
    - uses: astral-sh/setup-uv@v4
    - run: npm ci
      shell: bash
    - run: uv sync
      shell: bash
    - run: npm run build
      shell: bash
      if: inputs.build == 'true'
```

Note: The composite action includes `actions/checkout@v4` as its first step. This means callers use `uses: ./.github/actions/setup` directly without a separate checkout step.

**Job 4: `smoke-test` (GATE-03)**
- Uses the composite action: `uses: ./.github/actions/setup` (build: true, needs compiled dist/index.js)
- Sends the MCP initialize JSON-RPC request to the server via stdin pipe and validates the response
- CRITICAL: Must set dummy credentials as env vars to pass startup validation:
  ```yaml
  env:
    ZLIBRARY_EMAIL: "ci@test.com"
    ZLIBRARY_PASSWORD: "ci-test-password"
  ```
- Implementation:
  ```bash
  INIT_REQUEST='{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"ci-smoke-test","version":"0.0.0"}}}'
  INIT_NOTIFICATION='{"jsonrpc":"2.0","method":"notifications/initialized"}'
  RESPONSE=$(printf '%s\n%s\n' "$INIT_REQUEST" "$INIT_NOTIFICATION" | timeout 15 node dist/index.js 2>/dev/null | head -1)
  echo "$RESPONSE" | jq -e '.jsonrpc == "2.0" and .id == 1 and .result.protocolVersion != null and .result.serverInfo.name == "zlibrary-mcp" and .result.capabilities.tools != null'
  ```
- The `timeout 15` prevents hanging if the server doesn't exit after stdin closes
- The `2>/dev/null` suppresses stderr (debug messages, credential warnings)
- `jq -e` exits non-zero if the expression evaluates to false or null
- `head -1` captures only the first line of stdout (the JSON-RPC response)
- The pipe closing stdin causes the MCP StdioServerTransport to trigger server shutdown

**Job 5: `docker` (GATE-04)**
- Runs ONLY on push to master, NOT on PRs (Docker builds are slow, ~2-5 min):

- Does NOT attempt full health check (that requires real Z-Library credentials via SuperGateway). The goal is verifying the image builds and the Node.js runtime works inside it.
